# OpenCart Manual Testing Project

## Project Overview

This project demonstrates manual testing of the OpenCart e-commerce application. It focuses on validating core functionalities and ensuring the system behaves as expected under different user scenarios.

## Objective

The objective of this project is to verify that the application is functionally correct, user-friendly, and free from major defects that could impact the user experience.

## Documents Included

* **Test Plan:** Defines the testing strategy, scope, and approach
* **Test Cases:** Covers functional and edge-case scenarios for key modules
* **Bug Reports:** Contains defects identified during testing with proper details

## Testing Scope

The following modules are covered as part of testing:

* User Registration & Login
* Product Search & Filtering
* Product Details Page
* Shopping Cart
* Checkout Process
* Order Management
* Admin Panel (basic operations)

## Tools Used

* Manual Testing
* JIRA (for defect tracking)
* Microsoft Excel (for test case management)

## Key Highlights

* Designed and executed multiple test scenarios across different modules
* Identified defects related to validation and login functionality
* Performed functional and regression testing to ensure stability
